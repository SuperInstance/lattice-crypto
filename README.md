# lattice-crypto

A pure-Rust library implementing **lattice-based cryptography primitives**, including
LLL basis reduction, approximate shortest vector computation, and Learning With Errors
(LWE) key generation, encryption, and decryption.

## Why This Matters

Lattice-based cryptography is the leading candidate for **post-quantum cryptography**.
Unlike RSA and elliptic curve cryptosystems, which are vulnerable to Shor's algorithm
on a quantum computer, lattice problems (Shortest Vector Problem, Learning With Errors)
are believed to be hard even for quantum adversaries. NIST has already standardized
lattice-based schemes (Kyber/ML-KEM for key encapsulation, Dilithium/ML-DSA for
signatures).

This library provides clean implementations of the fundamental building blocks,
making it ideal for learning about post-quantum cryptography and for prototyping
new lattice-based protocols.

## Features

- **LatticePoint**: n-dimensional integer vectors with arithmetic operations
- **LatticeBasis**: matrix of basis vectors with Gram-Schmidt orthogonalization
- **LLL reduction**: Lenstra-Lenstra-Lovász algorithm for basis reduction
- **Shortest Vector Problem**: approximate solution via LLL reduction
- **LWE key generation**: create public/secret key pairs
- **LWE encryption/decryption**: encrypt and decrypt single-bit messages
- **Gram determinant**: compute the determinant of the Gram matrix

## Mathematical Background

### Lattices

A **lattice** L is a discrete subgroup of ℝⁿ, generated by integer linear
combinations of basis vectors:

```
L = { Σ zᵢ bᵢ  |  zᵢ ∈ ℤ }
```

where {b₁, ..., bₙ} is a basis of linearly independent vectors.

### Gram-Schmidt Orthogonalization

Given a basis B = {b₁, ..., bₙ}, the Gram-Schmidt process produces orthogonal
vectors {b₁*, ..., bₙ*}:

```
b₁* = b₁
bᵢ* = bᵢ - Σⱼ₌₁ⁱ⁻¹ μᵢⱼ bⱼ*
```

where μᵢⱼ = ⟨bᵢ, bⱼ*⟩ / ⟨bⱼ*, bⱼ*⟩.

### LLL Reduction

The LLL algorithm finds a "good" basis with short, nearly orthogonal vectors.
A basis is LLL-reduced with parameter δ (typically δ = 3/4) if:

1. **Size-reduced**: |μᵢⱼ| ≤ 1/2 for all i > j
2. **Lovász condition**: δ · ‖b*ₖ₋₁‖² ≤ ‖b*ₖ‖² + μ²ₖ,ₖ₋₁ · ‖b*ₖ₋₁‖²

LLL runs in polynomial time and guarantees the first basis vector satisfies:

```
‖b₁‖ ≤ 2^((n-1)/4) · det(L)^(1/n)
```

### Learning With Errors (LWE)

The LWE problem, introduced by Regev (2005): given (a, b = ⟨a, s⟩ + e mod q)
for random a, secret s, and small error e, find s. This is believed to be
as hard as worst-case lattice problems.

**Key generation:**
- Secret s ∈ ℤ_qⁿ
- Public key: matrix A ∈ ℤ_q^(m×n), vector b = As + e mod q (small error e)

**Encryption (bit m):**
- Choose random subset of rows
- Sum A rows → a, sum b values → b'
- Compute: c = b' + m · ⌊q/2⌋ mod q
- Ciphertext: (a, c)

**Decryption:**
- Compute c - ⟨a, s⟩ mod q
- If closer to 0 → message is 0; if closer to q/2 → message is 1

## Usage

```toml
[dependencies]
lattice-crypto = "0.1.0"
```

### Lattice Basis Operations

```rust
use lattice_crypto::{LatticePoint, LatticeBasis};

let basis = LatticeBasis::new(vec![
    LatticePoint::new(vec![100, 1]),
    LatticePoint::new(vec![50, 50]),
]);

// Gram-Schmidt orthogonalization
let (ortho, mu) = basis.gram_schmidt();

// LLL reduction
let reduced = basis.lll_reduce(0.75);
println!("Reduced basis: {:?}", reduced.vectors);

// Find approximate shortest vector
let sv = basis.shortest_vector();
println!("Shortest vector: {} (norm = {})", sv, sv.norm());
```

### LWE Encryption

```rust
use lattice_crypto::{lwe_keygen, lwe_encrypt, lwe_decrypt};

// Generate key pair: n=4 dimensions, q=97 modulus, m=6 samples, error ±1
let (secret, public) = lwe_keygen(4, 97, 6, 1);

// Encrypt message bit
let ciphertext = lwe_encrypt(&public, 1, 1);

// Decrypt
let decrypted = lwe_decrypt(&secret, &ciphertext);
assert_eq!(decrypted, 1);
```

### Vector Arithmetic

```rust
use lattice_crypto::LatticePoint;

let a = LatticePoint::new(vec![1, 2, 3]);
let b = LatticePoint::new(vec![4, 5, 6]);

println!("Dot product: {}", a.dot(&b));     // 32
println!("a - b = {}", a.sub(&b));           // [-3, -3, -3]
println!("3a = {}", a.scale(3));             // [3, 6, 9]
println!("|a| = {}", a.norm());              // 3.7416...
```

## API Reference

| Type/Function | Description |
|---|---|
| `LatticePoint` | n-dimensional integer vector |
| `LatticePoint::dot()` | Dot product |
| `LatticePoint::norm()` | Euclidean norm |
| `LatticePoint::sub()` | Vector subtraction |
| `LatticePoint::scale()` | Scalar multiplication |
| `LatticeBasis` | Matrix of basis vectors |
| `LatticeBasis::gram_schmidt()` | Gram-Schmidt orthogonalization |
| `LatticeBasis::lll_reduce(delta)` | LLL basis reduction |
| `LatticeBasis::shortest_vector()` | Approximate SVP via LLL |
| `lwe_keygen(n, q, m, e)` | Generate LWE key pair |
| `lwe_encrypt(pk, msg, e)` | Encrypt a bit |
| `lwe_decrypt(sk, ct)` | Decrypt a ciphertext |
| `gram_determinant(basis)` | Det(G) where G = B^T B |

## Security Note

This library uses a simple deterministic PRNG for reproducibility and testing.
**Do not use in production.** Replace with a cryptographically secure RNG
(e.g., `rand::rngs::OsRng`) for any real-world application.
